# lab4_rbtree

* **3 вариант** — реализация красно-чёрного дерева.

### Решение
В ходе лабораторной работы было реализовано красно-чёрное дерево на Си. Необходимо было написать функции создания и вставки узла, поиска узла и удаления узла. При этом каждая функция подразумевала обязательную балансировку.

#### Структуры данных
Для подготовки к написанию функций было создано две структуры:
1. **Структура ноды**: Состоит из текстового ключа длиной не более 7 символов, числового значения, цвета и ссылок на 2 потомков и родителя. Тут родитель необходим для удобной балансировки.
2. **Структура дерева**: Содержит указатель на корень и указатель на листовую вершину `NIL`. Это необходимо, чтобы избежать проверок на ноль и не получить исключений. То есть мы каждый раз знаем, где в памяти хранится определенная нода и корень.

#### Разработка функций
* **Создание дерева**: Функция, в которой мы выделяем память на дерево и на узел в нём и заполняем значения по умолчанию `NIL`, чтобы в будущем знать, какая вершина листовая. Также соблюдено условие, что все листья чёрные, значит дефолтно устанавливаем цвет чёрный.
* **Создание ноды**: Включает в себя выделение памяти на ноду и заполнение всех ячеек памяти в структуре. При этом вставляемая вершина всегда красная.
* **Функция вставки (`insert`)**: Работает по алгоритму бинарного поиска. Она сравнивает значения ключей и выбирает, в каком поддереве искать место для ноды. Далее вызывается функция создания вершины.
* **Балансировка вставки**: После вставки необходимо провести балансировку. Мы циклично балансируем дерево до тех пор, пока родитель красный, ведь это нарушает правила. Рассматриваются случаи: красный дядя, линия и треугольник. Созданы две функции для поворота дерева: левый и правый поворот.
* **Функция удаления**: Сначала удаляет узел как в обычном BST (замена на преемника или ребёнка), а затем при необходимости запускает балансировку `fix_delete`. Балансировка восстанавливает свойства дерева через перекраску и повороты, устраняя «двойной чёрный» и сохраняя баланс.
* **Печать**: Мы выводим значения начиная с правого поддерева и указываем цвет для наглядности.
* **Очистка памяти**: Написаны функции для рекурсивного очищения памяти.

#### Интерфейс и тесты
Главный файл содержит **CLI-интерфейс** для удобного тестирования и возможности запуска программы из файла. Также создан **Makefile**, чтобы не пересобирать программу каждый раз заново, а изменять лишь переписанные файлы.
Реализованы юнит-тесты для:
* Пустого дерева;
* Вставки 1 элемента;
* Вставки нескольких элементов;
* Поиска и удаления.
